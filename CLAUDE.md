# Team Prompts — CLAUDE.md

Internal team tool: generate YouTube video metadata (titles, descriptions, copy) from video
**scripts** using a shared OpenRouter key. The **script** is the single organizing key — there are
**no categories**. Pick a script → center shows everything ever generated for it, with attribution.

## Golden rules

- **Use `bun`, never `npm`/`yarn`/`pnpm`** for the frontend. Every command: `bun ...`.
- **Add shadcn/ui components via the shadcn MCP** — do not hand-write shadcn primitives.
- **Fetch current docs via Context7 MCP** before coding against any framework/library — training
  data may be stale. Treat Context7 output as authoritative for version-specific APIs.
- **The OpenAPI document is the FE/BE contract.** The frontend API client is **generated by orval**
  from the backend's OpenAPI doc — never hand-write fetch calls. When the backend changes the API,
  re-run orval (`bun run gen:api`) and update consumers.
- **Center map design** comes from claude_design ("Team Prompts - Map (shadcn)") — implement it,
  don't invent a layout. (Needs `/design-login` for the claude_design MCP.)

## Stack (pinned, do not substitute)

**Backend** — .NET 10 / C# Web API · ASP.NET Core Identity (email login) + policy-based RBAC ·
EF Core + Npgsql · SignalR (token streaming) · Hangfire on PostgreSQL (batch jobs) · Polly
(OpenRouter resilience) · FluentValidation · Serilog · UglyToad.PdfPig (PDF text extraction).

**Frontend** — Next.js (App Router) + Bun · shadcn/ui · TanStack Query v5 + TanStack Table ·
React Hook Form + Zod · next-themes · dnd-kit · `@xyflow/react` (only if the design is node/edge
based) · `react-diff-viewer-continued` (word-level prompt diff) · **orval** (OpenAPI → TanStack
Query hooks).

**DB** Postgres. **Infra** Docker Compose + Caddy reverse proxy (proxies `/api/*`→api, `/*`→web,
upgrades SignalR WebSocket). **AI** OpenRouter (default model `openai/gpt-5`, confirmed at runtime
via the models endpoint; selectable per generation).

## Backend layering (clean architecture)

```
TeamPrompts.Domain         entities, enums, IFileStorage — no external deps
TeamPrompts.Application    services, DTOs, FluentValidation, OpenRouter client interface
TeamPrompts.Infrastructure EF Core DbContext, Npgsql, migrations, Identity, file storage impl, OpenRouter HTTP client (Polly)
TeamPrompts.Api            endpoints, SignalR hub, Hangfire wiring, Serilog, Swagger/OpenAPI emission
```

Thin controllers/endpoints; logic in Application services; FluentValidation on all inbound DTOs;
async all the way; Serilog (console + rolling file).

## Frontend conventions

App Router. **Server state ONLY through TanStack Query + orval-generated hooks** — no inline
fetch. Forms via RHF + Zod (no native `<form>` submit semantics fighting). shadcn for all UI.
4-zone layout: left Scripts · right Prompts · center Map · bottom Tray (shadcn `resizable`).

## Domain model (no categories)

- **AppUser** : IdentityUser — `DisplayName`. Email = login id. **5 tiered roles** via policies:
  `Owner`⊇`Admin`⊇`PromptEditor`⊇`Member`⊇`Viewer` (policies `Admin`/`PromptEditor`/`Member`; Viewer holds
  none = read-only). Model choice, prompt-content viewing/editing, and all deletes are role-gated.
- **Script** — `Name`, `OriginalFileName`, `FileType` (Pdf|Txt), `ExtractedText`, `StorageKey`
  (original file via IFileStorage), `CreatedByUserId`, timestamps.
- **Prompt** — `Name`, `CreatedByUserId`, `MainVersionId` → PromptVersion.
- **PromptVersion** — `PromptId`, `ParentVersionId?` (branching), `Content`, `AuthorUserId`,
  `Note`, `IsMain`, `CreatedAt`. Promote-to-Main repoints `Prompt.MainVersionId` + flips `IsMain`.
- **GenerationRun** — batch grouping: `CreatedByUserId`, `Status`. Single-shot runs may be null.
- **GenerationSession** — one prompt-version run vs one script: `RunId?`, `ScriptId`, `PromptId`,
  `PromptVersionId`, `Model`, `Status` (Queued|Streaming|Completed|Failed), `CreatedByUserId`.
- **GenerationResult** — `SessionId`, `Index`, `Content`, `Kind?` (Title|Description|Hook|Tags|
  Thumbnail | null=freeform). ~4–6 per session.
- **ResultFavorite** — (`GenerationResultId`, `UserId`). Tray = user's favorites for active script.
- **ResultCopyEvent** — (`GenerationResultId`, `UserId`) copy attribution.
- **AppSettings** — single row: `OpenRouterApiKeyEncrypted` (write-only, never returned),
  `DefaultModel`, `AvailableModels` (json).

## Assumptions (decisions made during build)

- **Auth = cookie-based** ASP.NET Identity. FE + API are same-origin behind Caddy, so cookies
  (SameSite=Lax) avoid token storage/XSS concerns. Public self-registration **disabled**; an
  admin-only endpoint creates accounts. Login/logout/me endpoints are custom (not the raw
  `AddIdentityApiEndpoints` register flow).
- **IFileStorage stores the ORIGINAL uploaded file bytes** (the swappable blob seam → MinIO later).
  Extracted text lives on `Script.ExtractedText` (canonical, searchable). `Script.StorageKey` points
  at the stored original. DB-backed impl `PostgresFileStorage` writes to a `StoredFile` table.
- **API key encryption** = ASP.NET Core Data Protection (`IDataProtector`), key ring persisted to a
  mounted volume (`DATA_PROTECTION_KEYS` path). Never returned by any GET.
- **Variants per session = 5** (in the 4–6 range), constant `GenerationDefaults.VariantCount`.
- **Hangfire.PostgreSql** configured via `UsePostgreSqlStorage(o => o.UseNpgsqlConnection(cs))`;
  dashboard gated to Admin policy.
- **orval** mode `tags-split`, client `react-query`, custom axios mutator (sends cookies,
  `withCredentials`). Regenerate with `bun run gen:api`.
- **OpenRouter model id** for "GPT-5" confirmed at runtime via `/models`; default seeded as
  `openai/gpt-5`, overridable in Settings.
- EF Core migrations applied on API startup (idempotent `Database.Migrate()`), guarded for dev.

## Commands

- Backend: `dotnet build backend/TeamPrompts.slnx` · run API `dotnet run --project backend/src/TeamPrompts.Api`
  (solution is `.slnx`, not `.sln`). Migrations: `dotnet ef migrations add <Name> -p src/TeamPrompts.Infrastructure -s src/TeamPrompts.Infrastructure -o Persistence/Migrations` (a design-time factory makes this work without the API).
- Frontend: `cd frontend && bun install` · `bun run dev` · `bun run build` · `bun run gen:api`
- Refresh the API contract: run the API, then `curl http://localhost:5258/openapi/v1.json -o backend/openapi.json` and `cd frontend && bun run gen:api`.
- Full stack: `docker compose up --build` (db + api + web + caddy). App at http://localhost. Admin seeded from `.env`.

## Gotchas discovered during the build (don't re-trip these)

- **OpenAPI version**: the API is forced to emit **OpenAPI 3.0** (`AddOpenApi(o => o.OpenApiVersion = OpenApi3_0)`). orval mis-detects query vs mutation against .NET's default 3.1 output (turns GETs into mutations). Keep 3.0.
- **orval int/enum types**: .NET's 3.0 writer omits `type` on integer and enum schemas. `orval.config.ts` has an input `transformer` that restores `type: integer` / `type: string` so the client isn't `unknown`. Enum string values come from `[JsonConverter(typeof(JsonStringEnumConverter))]` on the Domain enums.
- **orval query override**: do NOT set `override.query.useQuery/useMutation` — it forces every operation (incl. POST) to a query hook. Omit it; orval splits by HTTP method (GET→useQuery, others→useMutation).
- **Next.js 16 + shadcn (Base UI registry)**: components use the Base UI **`render` prop**, not `asChild` (`<Button render={<Link/>}>`, `<DialogTrigger render={<Button/>}>`). Tooltip uses `delay` not `delayDuration`. `Select` `onValueChange` yields `string | null`. `react-resizable-panels` v4 uses `orientation` (not `direction`). `middleware.ts` is renamed `proxy.ts` (we don't use it — auth is gated client-side).
- **Script upload** is hand-written (`src/lib/api/uploads.ts`) because .NET models `IFormFile` poorly in OpenAPI; the generated upload body is unusable.
- **claude_design import was deferred**: the claude_design MCP wasn't connected (needs `/design-login`). The center block-map is built with shadcn + CSS grid per the spec's stated fallback. To match the exact Claude Design, run `/design-login`, import `Team Prompts - Map (shadcn)`, and refit `generation-map.tsx` / `session-block.tsx` / `result-card.tsx`.
- **Batch script selection** uses checkboxes (clearer than drag); dnd-kit is used for tray reordering. The bottom tray = the current user's favorited results for the active script ("star" a result = add to tray).
